# 스프링 계층형 아키텍처
    시스템의 기능을 여러 계층으로 나눈 것

    Browser: 데이터를 렌더링
    Controller: Service 계층의 비즈니스 로직 이용
    Service: 비즈니스 로직이 담기는 계층
    DAO: 데이터 접근 객체
    DB: 데이터베이스

    DTO: 데이터 전송 객체로, 각 계층끼리 필요한 정보만 전달
    Entity: DB와 맵핑되는 핵심 객체로, 외부 노출 X

# 패키지 구조
    계층형 구조: app을 기능별(Controller, Service etc.)로 나눈다.
    도메인형 구조: 모든 클래스를 한 곳에 모아 둔다. 쉬운 코드 탐색

# 비즈니스 로직
    데이터 생성, 표시, 저장 등 프로그램이 무엇을 해야 하는지 정의하는 규칙

    원자성: 하나의 method에서 모든 로직이 실행되거나 실행되지 않거나 둘 중 하나만 가능하여, 더 이상 쪼갤 수 없는 성질

# 상태 코드
    200 OK: 처리 성공
    201 Created: 데이터 생성 성공
    204 No Content: 요청은 성공이지만 내용 없음
    400 Bad Request: 클라이언트 요청 오류
    401 Not Found: 요청 데이터 없음
    500 Internal Server Error: 서버 에러

# 스프링
    Java 백엔드 애플리케이션 프레임워크
    
    스프링 빈: 애플리케이션 전역에 사용할 공용 객체
    스프링 컨테이너: 빈을 저장하고 사용한다. 필요한 빈은 프레임워크가 자동으로 가져와준다.

    등록: 
    1. @ComponentScan: 어떤 클래스가 빈인지 찾아 등록
    2. @Component: 해당 클래스를 빈으로 표시

# 의존성 주입
    객체를 직접 생성하지 않고 외부에서 주입한다. 효율적인 메모리 사용

    생성자 주입 방법: 변수를 final로 선언하여 값의 변경을 막음.
