# 레포지토리(Repository) 계층
    데이터베이스에 접근하여 CRUD(Create, Read, Update, Delete) 작업을 수행하는 계층

# ORM
    객체이서 DB에 접근할 때 반복적인 SQL을 일일히 다루는 것 대신 데이터를 다종으로 매핑

# 엔티티 매니저
    ORM의 기능을 엔티티 매니저가 제공하여 SQL을 작성하지 않고 CRUD 작업이 가능
    표준 모델인 JPA를 사용
    
    영속: 엔티티를 영속성 컨텍스트에 저장된 상태
    비영속: 영속과 무관한 상태
    준영속: 영속성 컨텍스트에 저장되었다가 분리된 상태
    삭제: 영속성 컨텍스트에 '엔티티 삭제'로 기록한 상태

# 영속성 컨텍스트
    JPA에서 엔티티를 관리하기 위해 만들어진 가상 공간
    
    트랜잭션을 지원하는 쓰기 지연: 변경사항을 모아두었다가 한 번에 전달
    1차 캐시: 한 번 조회한 객체를 내부에 저장하고 재사용
    동일성 보장: 같은 엔티테를 여러번 조회해도 항상 동일한 객체임을 보장
    변경 감지: 영속 상태의 엔티티의 변경 사항을 자동으로 감지

# JPA 동작 방식
    등록: 영속성 컨텍스트에 저장한 뒤 DB에 동기화
    조회: DB에 조회하여 영속성 컨텍스트로 가져온 뒤 반환
    수정: 엔티티를 조회한 뒤 영속성 컨텍스트에서 수정 후 변경이 감지되면 DB에 동기화
    삭제: 엔티티를 조회한 뒤 영속성 컨텍스트에서 삭제로 분류한 후 DB에 동기화하여 삭제
